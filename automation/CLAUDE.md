# 自動化部署

## 役割

ナレッジソサエティの業務で発生する **繰り返し作業のスクリプト化** を担う部署。

「自分で手を動かせば終わるが、面倒で手が止まりがち」な作業を Python / PowerShell 等のスクリプトに置き換える。

## 自動化したい候補（既出のもの）

| 案件 | 内容 | 状態 |
|---|---|---|
| アフィリ月次レポ自動化 | ASP 管理画面から CSV を自動 DL → Google Drive 格納 → Chatwork 通知 | 検討中 |
| 画像差し替え自動化 | 銀行手数料変更等の画像更新を、Nano Banana 等の AI 画像編集と組み合わせて半自動化 | 検討中 |
| サテライト記事一括投稿 | 80〜90% 品質の記事を WP 4 サイトに分散投稿 | 検討中 |
| SNS 投稿自動化 | 動画 + キャプションを Instagram / TikTok 等に予約投稿 | 検討中 |

※ 状態欄は運用状況に合わせて編集してください。

## できること（Claude への呼びかけ例）

### スクリプト設計

- 「{ASP 名} の管理画面から月次 CSV を DL → Drive にアップロード → Chatwork 通知するスクリプトの構成を設計して。使う言語は Python」
- 「画像差し替えタスクを、フォルダ監視 + AI 画像生成 + WP 自動更新で半自動化したい。要件定義書を作って」

### スクリプト実装

- 「{スクリプト名} を実装して。動作確認用のサンプルデータも一緒に」
- 「このスクリプトを Windows のタスクスケジューラに登録する手順書を作って」

### エラー対応

- 「昨日動かした {スクリプト名} がエラーで止まった。ログを見て原因を特定して」
- 「{エラーメッセージ} が出た時の対処法を教えて」

### メンテナンス

- 「{ASP 名} の管理画面 UI が変わった。スクリプトのどこを直す必要がある？」

## 運用ルール

- ログイン情報・API キーは **このフォルダ内に直接書かない**（`.env` ファイル or Windows 資格情報マネージャ）
- スクリプトは `scripts/{用途}/{スクリプト名}.py` 形式
- 実行ログは `logs/{YYYY-MM-DD}-{スクリプト名}.log` に出力
- 大事なファイル操作（削除・上書き）は必ず確認プロンプトを挟む
- 1 つのスクリプト = 1 つの責務（あれもこれも詰め込まない）

## NG / 注意

- 認証情報を Git にコミットしない（`.gitignore` で `.env` を必ず除外）
- 本番データを直接書き換えるスクリプトは、まず **テスト環境** で動かす
- 外部 API のレート制限を超えないよう、適切な間隔を空ける

## フォルダ構成（例）

- `scripts/{用途}/` — 用途別スクリプト
  - `affiliate-report/` — アフィリ月次レポート関連
  - `image-update/` — 画像差し替え関連
  - `sns-post/` — SNS 投稿関連
- `templates/` — スクリプトのひな型
- `logs/` — 実行ログ
- `.env.example` — 環境変数のサンプル（実際の `.env` は git 管理外）

## 部署間連携

- アフィリレポート関連 → `affiliate/` と連携
- 記事投稿関連 → `seo/` と連携
- SNS 投稿関連 → `content/` と連携
