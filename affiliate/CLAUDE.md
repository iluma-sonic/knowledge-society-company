# アフィリ運用部署

## 役割

ナレッジソサエティの **金融系アフィリエイト運用** を担う部署。

ASP の月次成果レポートの整理・要約・前月比分析、案件画像の差し替え指示、訴求文のブラッシュアップを担当する。

## 利用 ASP

| ASP 名 | 主な取り扱い案件 | レポート形式 |
|---|---|---|
| {ASP 1} | {案件カテゴリ} | CSV / 管理画面 |
| {ASP 2} | {案件カテゴリ} | CSV / 管理画面 |
| {ASP 3} | {案件カテゴリ} | CSV / 管理画面 |

※ ASP 名・取り扱い案件は運用状況に合わせて編集してください。

## できること（Claude への呼びかけ例）

### 月次レポート整理

- 「今月の {ASP 名} の成果レポート（CSV）を整理して。前月比・案件別・サイト別に分けて」
- 「整理した結果を、{担当者名} に共有する用の Markdown サマリにして」
- 「先月と比べて伸びた案件 / 落ちた案件のトップ 3 をピックアップして、理由の仮説を 2 つずつ」

### 画像差し替え指示

- 「{案件名} の銀行手数料が {旧 → 新} に変更になった。サイト内で差し替えが必要な画像を一覧化して」
- 「画像差し替え時の指示書を、デザイナーさんに渡す形でまとめて」

### 訴求文ブラッシュアップ

- 「{案件名} のバナー下訴求文を 3 案。ターゲット別（初心者 / 中級者 / 比較検討中）」
- 「この訴求文、ちょっと煽りが強いのでソフトに直して」

### キャンペーン管理

- 「今月の {ASP 名} の特別案件・キャンペーンをリストアップ」
- 「{キャンペーン} の終了日と、その後の差し替えタスクを `secretary/todos/` に登録」

## 運用ルール

- ASP ログイン情報・売上数値の生データは **このフォルダ内に記入しない**（環境変数 or 別管理）
- 月次レポートの CSV は `monthly-reports/{YYYY-MM}/` に保管
- 整理サマリは `monthly-summaries/{YYYY-MM}.md` に Markdown で出力
- 案件画像の差し替え指示は `image-update-tasks/{YYYY-MM-DD}-{案件名}.md` に記録

## NG / トーン

- 「絶対儲かる」「リスクゼロ」等の断定表現
- 競合他 ASP・他案件の名指し批判
- 出典のない数字
- 個人情報・売上金額の外部送信

## フォルダ構成（例）

- `monthly-reports/{YYYY-MM}/` — ASP からダウンロードした CSV 等の生データ
- `monthly-summaries/{YYYY-MM}.md` — 整理済みサマリ
- `image-update-tasks/` — 画像差し替えタスク管理
- `campaigns/` — キャンペーン情報
- `_archive/` — 終了案件アーカイブ

## 部署間連携

- 月次レポート自動 DL → `automation/` に依頼
- アフィリ訴求記事のリライト → `seo/` に依頼
- 動画での案件紹介 → `content/` に依頼

---

> **カスタマイズしてください**:
> ASP 名・案件カテゴリ・主要担当者の連絡フロー等を埋めていくと、月次運用がスムーズになります。
