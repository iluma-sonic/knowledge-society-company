# 営業部署

## 役割

新規開拓・既存顧客対応・提案・見積・受注管理・渉外（提携交渉・スポンサー獲得）を担う部署。

「**お客様 / 取引先との関係を作って、お金が動く前のすべての段取り**」を扱う。

## できること（Claude への呼びかけ例）

### リード・顧客リスト管理

- 「見込みリストを整理して。直近のやり取りで優先度が高い 5 件を出して」
- 「{顧客名} の過去のやり取りをまとめて、次回連絡時のリマインダーを作って」
- 「先月入った見込みリードの一覧をテーブル形式で」

### 提案資料・見積書

- 「{顧客名} 向けの提案書骨子を作って。先方の課題は {内容}」
- 「見積書のテンプレを使って、{プラン名} の金額シミュレーションを 3 パターン作って」
- 「提案資料の表紙・章立てだけ先に組んでみて」

### 商談・面談の準備

- 「明日の {顧客名} との商談用に、先方のサイト情報・業界トレンドをリサーチして」
- 「商談の議題と想定質問・想定回答のメモを作って」
- 「面談後の議事録テンプレを作って」

### 渉外・提携

- 「{業界} の提携候補リストを作って。先方のメリット / こちらのメリットを整理」
- 「{相手先} へのアプローチ文面を 3 トーン（フォーマル / カジュアル / 短文）で」

### 受注後の引き継ぎ

- 「{案件名} の受注が決まった。`production/` に渡す引き継ぎメモを作って」

## 運用ルール

- 顧客情報（氏名・連絡先・取引金額）は **このフォルダ内に直接書かない**（別管理 or 環境変数）
- 顧客別フォルダ命名: `clients/{client-name}/`
- 商談メモ: `meetings/{YYYY-MM-DD}-{client-name}.md`
- 見積書: `quotes/{YYYY-MM-DD}-{client-name}-{slug}.md`

## NG / トーン

- 競合他社の名指し批判
- 数字の根拠なき断定（「絶対に売上が上がります」等）
- 価格交渉時の安易な値引き提案（一律ルール化禁止、ケースバイケース）

## フォルダ構成（例）

- `clients/` — 顧客別フォルダ
- `quotes/` — 見積書
- `proposals/` — 提案資料
- `meetings/` — 商談・面談メモ
- `templates/` — 各種テンプレート
- `_archive/` — 終了案件アーカイブ

## 部署間連携

- 受注後の制作依頼 → `production/` に引き継ぎ
- 既存顧客の定常運用 → `operations/` と連携
- 重要な提案方針の意思決定 → `secretary/decisions/` に記録

## ナレッジソサエティ業務での想定例

最初はナレッジソサエティ様の以下の業務に使えます（運用しながらカスタマイズしてください）:

- バーチャルオフィス契約者対応・契約更新
- アフィリ ASP との提携交渉・新規案件の獲得
- 媒体スポンサー候補のリサーチ
- 法人向け提案資料の作成
